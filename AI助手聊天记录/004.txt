## 代码整体架构分析

### **核心转换流程**

这个脚本将PDF转换为ePub的流程分为三个阶段：

1. **PDF信息提取阶段**
   ```bash
   pdfinfo $1              # 获取PDF元数据（标题、作者）
   pdffonts $1             # 获取PDF中的字体信息
   ```

2. **PDF → HTML转换阶段**（关键步骤）
   ```bash
   pdf2htmlEX --embed-css 0 --embed-font 0 --embed-image 0 \
              --split-pages 1 --bg-format $imgformat \
              --dpi $dpi --fit-width $hres --fit-height $vres \
              --page-filename mybook%04d.page \
              --css-filename mybook.css mybook.pdf
   ```

3. **HTML → ePub打包阶段**
   创建ePub的标准目录结构并压缩

## 详细分析结果 🔍

我已经详细研究了生成的ePub文件和原始PDF，找到了**加粗文本丢失的根本原因**：

### **问题诊断**

#### 1️⃣ **PDF中的字体结构**
PDF文件中包含的字体列表：
- ✅ `MicrosoftYaHei` - 普通文本字体
- ✅ `MicrosoftYaHei-Bold` - **单独的粗体字体**
- ✅ `SimHei`、`SimSun`、`FangSong` 等

**关键点**：PDF通过使用**不同的字体文件**来表现文本样式，而不是使用CSS的 `font-weight` 属性。

#### 2️⃣ **ePub中的CSS样式分析**
生成的ePub文件中的所有字体定义：

```css
.ff1 { font-family:ff1; font-weight:normal; ... }
.ff2 { font-family:ff2; font-weight:normal; ... }
.ff3 { font-family:ff3; font-weight:normal; ... }
.ff4 { font-family:ff4; font-weight:normal; ... }
.ff5 { font-family:ff5; font-weight:normal; ... }
...
.ffe { font-family:ffe; font-weight:normal; ... }
```

**问题**：**所有字体类都被设置为 `font-weight:normal`**，没有任何 `font-weight:bold` 的声明。

### **根本原因** 💡

```
PDF处理链：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
粗体文本     → 使用"MicrosoftYaHei-Bold"字体
              ↓
          pdf2htmlEX转换
              ↓
          HTML <span class="ff5">...</span>
              ↓
          CSS: .ff5 { font-weight:normal; }
              ↓
          ❌ 加粗信息丢失！
```

### **为什么会丢失？**

1. **字体名称识别失败**
   - `pdf2htmlEX` 不会解析字体名称中的 `-Bold` 后缀
   - 它只是将字体文件转换为CSS类

2. **无法从字体文件推断样式**
   - 虽然 `MicrosoftYaHei-Bold` 是粗体字体文件
   - 但 `pdf2htmlEX` 不能自动检测并设置 `font-weight:bold`

3. **固定布局转换方式**
   - pdf2epubEX 脚本使用固定布局ePub
   - 主要目的是保留PDF的页面布局，而非提取文本语义

### **转换流程中的关键参数**
```bash
pdf2htmlEX --embed-font 0 \           # 不内嵌字体
           --dpi 150 \
           --split-pages 1 \          # 分页处理
           mybook.pdf
```

参数 `--embed-font 0` 表示不直接嵌入字体，而是通过 `@font-face` 引用，但这个过程中丢失了样式信息。

### **结论**

这是 `pdf2htmlEX` 工具的**设计限制**：
- 它将PDF字体映射到CSS类，但不处理字体权重（bold/normal）的区分
- 对于中文文档（使用字体变体表现样式），这个限制尤其明显
